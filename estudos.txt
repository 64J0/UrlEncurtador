EJS -> Template engine.
Effective JavaScript templating

  Um template engine permite a utilização de arquivos de template estáticos na sua aplicação. Em runtime, o templete engine substitui algumas variáveis no arquivo de template e transforma o template em um arquivo HTML para ser enviado ao cliente.
  Para renderizar arquivos de template, é necessário configurar algumas propriedades do Express. A primeira configuração indica onde estão as Views do código:
  app.set('views', './views');
  A segunda configuração indica qual o template engine que será usado:
  app.set('view engine', 'ejs');
  Então após essas configurações, deve-se instalar o pacote npm correspondente à view engine específica que será usada.

  Depois de especificar a view engine, não é necessário especificar a engine ou carregar o módulo da template engine no código, o Express fará isso internamente.

JavaScript export statement:

  O comando export é usado quando criamos módulos JavaScript e desejamos exportar funções, objetos ou valores primitivos desse módulo, para que esses trechos de código possam ser usados em outros programas.
  Existem dois tipos de exports:
  1. Exports nomeados (Zero ou mais exports por módulo)
  2. Default exports (Um por módulo)

  # No Node.js:
  module.exports ou exports é um objeto especial que é incluído em todo arquivo JS em uma aplicação Node.js por padrão. "module" é uma variável que representa o módulo atual e "exports" é um objeto que será exposto como módulo. Então, o que quer que você associe a module.exports ou exports, será exposto como um módulo.
